## Breaking Changes

- remote: Change `NewMessage(data, svcInfo, ri, msgType, rpcRole)` -> `NewMessage(data, ri, msgType, rpcRole)`; remove `NewMessageWithNewer` (#1818)
- remote: Remove `ServiceInfo()`, `SpecifyServiceInfo()`, `ProtocolInfo()`, `SetProtocolInfo()` from `Message` interface (#1818)
  Migration: resolve service info via rpcinfo instead.
- remote: Remove `ProtocolInfo` struct and `NewProtocolInfo` function (#1818)
- remote: Remove `FillSendMsgFromRecvMsg` function (#1818)
- remote: Change `ServiceSearcher.SearchService(svcName, methodName, strict)` -> `SearchService(svcName, methodName, strict, codecType)` (#1837)
- remote: Remove `ServerOption.TargetSvcInfo` field (#1840)
- remote/codec: Change `SetOrCheckMethodName(methodName, message)` -> `SetOrCheckMethodName(ctx, methodName, message)` (#1855)
- remote/trans: Remove `trans.GetMethodInfo` and `trans.GetDefaultSvcInfo` helper functions (#1818)
- serviceinfo: Change `ServiceInfo.MethodInfo(name string)` -> `ServiceInfo.MethodInfo(ctx context.Context, name string)` (#1837)
- serviceinfo: Change `ServiceInfo.GenericMethod` type from `func(name string) MethodInfo` -> `GenericMethodFunc` = `func(ctx, methodName) MethodInfo` (#1818)
- serviceinfo: Remove `GenericClientStreamingMethod`, `GenericServerStreamingMethod`, `GenericBidirectionalStreamingMethod`; add `PackageName`, `CombineServiceKey`, `CombineServiceName` (#1837)
- server: Remove `server.WithSupportedTransportsFunc(f)` option (#1839)
- server: Remove `server.WithServiceMiddleware(mw)` register option; `RegisterOptions.Middlewares` replaced by `RegisterOptions.IsUnknownService` (#1839)
- rpcinfo: Add `MethodInfo() serviceinfo.MethodInfo` to `Invocation` interface (#1818)
  Migration: implement on all `Invocation` implementations.
- rpcinfo: Add `SetMethodInfo(serviceinfo.MethodInfo)` to `InvocationSetter` interface (#1818)
  Migration: implement on all `InvocationSetter` implementations.
- retry: Change `RPCCallFunc` from `func(ctx, Retryer) (RPCInfo, interface{}, error)` -> `func(ctx, Retryer, request, response) (RPCInfo, error)` (#1814, #1799)
- retry: Change `Retryer.Do(ctx, rpcCall, firstRI, request)` -> `Do(ctx, rpcCall, firstRI, request, response)` (#1814, #1799)
- retry: Change `Container.WithRetryIfNeeded(ctx, callOptRetry, rpcCall, ri, request)` -> add `response` parameter (#1814, #1799)
- retry: Remove `CtxRespOp` constant (#1814, #1799)
- kerrors: Remove `kerrors.ErrRPCFinish` (#1799)
- ttstream: Rename `NewClientProvider` -> `NewCliTransHandlerFactory`; `ClientProviderOption` -> `ClientHandlerOption`; `ServerProviderOption` -> `ServerHandlerOption` (aliases kept) (#1805)
- genericclient: `ClientStreamingClient`, `ServerStreamingClient`, `BidiStreamingClient` no longer embed `streaming.ClientStream`; add explicit `Header()`, `Trailer()`, `CloseSend()`, `Context()` and new `Streaming() streaming.ClientStream` (#1837)
  Migration: use explicit methods or `Streaming()` to access underlying stream.

## Deprecations

- generic: `generic.BinaryThriftGeneric()` is deprecated; use `generic.BinaryThriftGenericV2(svcName string)` instead. (#1837)
- server: `server.WithGeneric(g)` is deprecated; use `generic.BinaryThriftGenericV2` or `genericserver.RegisterUnknownServiceOrMethodHandler` instead. (#1837)
- client/genericclient: `genericclient.NewStreamingClient` is deprecated; use `genericclient.NewClient` instead. `genericclient.StreamingServiceInfo` is deprecated; use `generic.ServiceInfoWithGeneric` instead. (#1837)

## Features

- generic: Add `generic.BinaryThriftGenericV2(svcName string)` with per-service binary thrift generic support and `thrift.RawReaderWriter`, `thrift.RawReader`, `thrift.RawWriter` types (#1837)
- generic: Add streaming server types `generic.ClientStreamingServer`, `generic.ServerStreamingServer`, `generic.BidiStreamingServer` interfaces and `generic.ServiceV2` struct (#1837)
- generic: Add `generic.ServiceInfoWithGeneric(g Generic)` to create `*serviceinfo.ServiceInfo` from a Generic instance (#1837)
- server/genericserver: Add `genericserver.NewServerV2(handler, g, opts...)`, `genericserver.RegisterService(svr, handler, g, opts...)`, `genericserver.RegisterUnknownServiceOrMethodHandler(svr, handler)` and `genericserver.NewUnknownServiceOrMethodServer(handler, opts...)` for handling unknown service/method traffic with `UnknownServiceOrMethodHandler` struct (#1837)
- ttstream: Support ctx cancel with detailed canceled error reporting via `contextWithCancelReason` and new error types `errBizCancel`, `errDownstreamCancel`, `errUpstreamCancel`, etc. (#1821)
- ttstream: Enhanced `Exception` type with `side`, `cancelPath`, and `cause` fields for better troubleshooting (#1821)
- grpc: Support dumping `MaxConcurrentStreams` of HTTP2 Client in `clientTransportDump` and log when stream creation is blocked (#1820)
- server: Support custom ctx key to pass to downstream via `consts.SERVICE_INLINE_CUSTOM_CTX_KEY` (#1709)
- rpcinfo: Add `Invocation.MethodInfo()` and `InvocationSetter.SetMethodInfo(methodInfo)` methods; add `InvocationServiceInfoKey` extra key (#1818)
- kerrors: Add `kerrors.ErrNonExistentMethod`, `kerrors.ErrNotStreamingMethod`, `kerrors.ErrNotUnaryMethod` error constructors; add `kerrors.ErrStreamingCanceled` sentinel (#1818, #1821)
- remote: Add `remote.ServiceSearcher` context helpers `remote.WithServiceSearcher(ctx, svcSearcher)` and `remote.GetServiceSearcher(ctx)` (#1818)
- diagnosis: Add `diagnosis.UnknownServiceKey` probe name (#1818)

## Fixes

- retry: Fix data race between timeout middleware and retry by using per-retry-call response objects and `shallowCopyResults` for response copying (#1814, #1799)
- retry: Fix panic not triggered when all retries return `ErrRPCFinish` (#1814)
- rpcinfo: Protect `bizErr` and `extra` fields of `invocation` by lock/atomic operations to fix concurrent read/write race (#1850)
- lbcache: Check existence before creating new Balancer in `BalancerFactory.Get` to prevent leakage (#1825)
- generic/descriptor: Fix `HTTPRequest.GetParam` nil pointer exception when `Params` is nil (#1827)
- generic/thrift: Fix integer range check in `writeInt16`/`writeInt32`/`writeInt64` to correctly handle negative values using `math.MinInt8`/`math.MaxInt8` bounds (#1861)
- ttstream: Fix `cancelFunc` panic when server-side stream receives Rst and then handler returns a customized error (#1859)
- client: Remove timer pool (`poolTimer`) to avoid timer race issue in `timeoutTask.Wait` (#1858)
- tool/kitex: Skip pb code gen for `-use` argument (#1819)
- tool/kitex: Disable fast API for protobuf (#1807)
- generic/thrift: Fix binary raw read nocopy error (#1857)

## Improvements

- perf: Optimize cost of setting service searcher by moving `ServiceSearcher` to context via `remote.WithServiceSearcher`/`remote.GetServiceSearcher` (#1855)
- ttstream/container: Lazy init cleaning task for `ObjectPool` to reduce goroutine overhead from creating many Generic Clients (#1842)
- remote: Remove string deepcopy in message handling (#1832)
- grpc: Access `metadata.MD` without `ToLower` (#1806)
- tool: Remove `kutils.StringDeepCopy` dependency from generated DeepCopy code (#1832)
