## Features

- generic: Add `generic.BinaryPbGeneric(svcName, packageName)` for raw protobuf binary payload generic calls (#1755, #1752)
- generic: Support TTHeader streaming generic calls with `EncodePayload`/`DecodePayload` handling `generic.Args`/`generic.Result` types (#1763)
- generic: Add `genericclient.Client.ClientStreaming`, `genericclient.Client.ServerStreaming`, `genericclient.Client.BidirectionalStreaming` methods for streaming generic calls (#1763, #1770)
- generic/proto: Add `proto.RawReaderWriter`, `proto.RawReader`, `proto.RawWriter` for binary protobuf streaming generic (#1755)
- ttstream: Process MetaFrame and reflect metadata to rpcinfo via `DecodeFrame`/`EncodeFrame` supporting `metaFrameType` (#1782)
- tool: Add `PatchFunc` type and `AppendExtensionPatches(f PatchFunc)` for thriftgo patcher extension; export `Patcher` type with `GetModule()` and `GetUtils()` methods (#1771)
- tool: Add `env.UsePrutalMarshal()` controlled by `KITEX_TOOL_USE_PRUTAL_MARSHAL` environment variable (#1759)
- client: Report error via tracer `DoFinish` when creating Stream fails (#1777)

## Fixes

- grpc: Inject current method name to rpcinfo via `context.WithValue(ctx, consts.CtxKeyMethod, methodName)` on server-side to fix `FROM_METHOD` missing in streaming handlers (#1792)
- ttstream: Fix server-side rpcinfo not set correctly causing metrics missing (#1787)
- generic: Fix enabling JSON mode of map generic not working by passing `forJSON` flag through `newMapThriftCodec(p, forJSON)` (#1778)
- generic/thrift: Align dynamicgo's write base behavior with old generic by adding `MergeBase` and `mergeBaseAny` functions (#1742)
- tool: Fix incorrect handling of proto file path on Windows in prutal by using `filepath.Base` (#1773)
- server: Fix trans server conn count race by using atomic operations and rewriting go net transport server with proper connection lifecycle (#1774, #1788)

## Improvements

- gonet: Rewrite go net transport server with proper connection lifecycle, idle timeout, read timeout, and graceful shutdown (#1788)
- grpc: Parse PayloadCodec from content subtype in server side via `getPayloadCodecFromContentType` and set it in rpcinfo config (#1783)
- ttstream: Log errors thrown by invoking handler (#1780)
- tool/prutal: Remove fastpb generation, truncate all fastpb files via `util.TruncateAllFastPBFiles` after processing (#1786)
- tool: Use `os.UserHomeDir()` to allow modifying OS env to set the git-cache dir (#1754)
