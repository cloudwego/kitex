## Breaking Changes

- retry: Change `Retryer.AppendErrMsgIfNeeded(err, ri, msg)` -> `AppendErrMsgIfNeeded(ctx, err, ri, msg)` (#1353)
- retry: Change `FailurePolicy.IsRespRetryNonNil`, `IsErrorRetryNonNil`, `IsRetryForTimeout` from value receiver to pointer receiver (#1353)
- retry: Change `BackupPolicy.String` from value receiver to pointer receiver (#1353)
- remote: Add `MallocAck(n int) error` to `NocopyWrite` interface (#1322)
  Migration: implement `MallocAck` on all `NocopyWrite` implementations.
- remote: Error message for unknown service changed from `UnknownMethod` to `UnknownService` with service name (#1368)
  Migration: update error message matching if used.
- remotecli: Change `NewStream` return from `(streaming.Stream, error)` -> `(streaming.Stream, *StreamConnManager, error)` (#1328)

## Deprecations

- retry: `ShouldResultRetry.ErrorRetry` deprecated in favor of `ShouldResultRetry.ErrorRetryWithCtx` (#1353)
- retry: `ShouldResultRetry.RespRetry` deprecated in favor of `ShouldResultRetry.RespRetryWithCtx` (#1353)
- retry: `FailurePolicy.IsErrorRetryNonNil` deprecated in favor of `FailurePolicy.IsErrorRetryWithCtxNonNil` (#1353)
- retry: `FailurePolicy.IsRespRetryNonNil` deprecated in favor of `FailurePolicy.IsRespRetryWithCtxNonNil` (#1353)
- loadbalance: `ConsistentHashOption.ExpireDuration` deprecated -- implementation no longer caches all keys (#1370)

## Features

- codec/thrift: Add `skipDecoder` enabling Frugal and FastCodec for standard Thrift Binary Protocol without payload length via `EnableSkipDecoder` (#1313)
- codec/thrift: Support fast write nocopy via `NocopyWrite` with netpoll link buffer using `WriteDirect`/`MallocAck` (#1322)
- circuitbreak: `NewCBSuite` accepts variadic `CBSuiteOption` for custom error type functions: `WithServiceGetErrorType`, `WithWrappedServiceGetErrorType`, `WithInstanceGetErrorType`, `WithWrappedInstanceGetErrorType` (#1257)
- retry: Add `ShouldResultRetry.ErrorRetryWithCtx` and `RespRetryWithCtx`; add `FailurePolicy.IsRespRetry`, `IsErrorRetry`, `ConvertResultRetry` (#1353)
- retry: Add `WithContainerCBSuiteOptions` to pass `CBSuiteOption` to retry circuit breaker (#1353)
- generic: Add `NewPbContentProviderWithDynamicGo` for parsing protobuf IDL from memory; add `PbFileProviderWithDynamicGo.UpdateIDL` (#1359)
- generic: Support `SetDynamicGoParseMode` for thrift/protobuf IDL providers (#1346)
- http: `NewDefaultResolver` accepts `WithIPv4()` and `WithIPv6()` options (#1352)
- registry: Add `Info.SkipListenAddr` field to prevent listen addr from overriding registered address (#1333)
- server: `WithPayloadCodec` now replaces codec by type instead of overriding all (#1362)
- mem: Add `SpanCache` linear allocator for batching small byte slice allocations to reduce GC pressure (#1276, #1314)
- bthrift: `Binary.WriteBinaryNocopy` uses `WriteDirect` for large payloads (>4KB); `Binary.ReadString`/`ReadBinary` use span cache allocator (#1322, #1276)
- tool: Add fast-codec support for Thrift Fieldmask (#1336)
- tool: Add dependencies compatibility checking (#1316)

## Fixes

- connpool: Reset `idleList` elements to nil in `pool.Get` and `pool.Evict` to prevent connection leak (#1307)
- netpollmux: Fix multi-service bug where first svcInfo was incorrectly assigned to `targetSvcInfo` (#1294)
- generic: Do not write generic method name for binary generic exception (#1308)
- generic: Fix "unknown service" error when using generic client by not writing IDLServiceName (#1379)
- codec: Fix `EnableSkipDecoder` switch not working in Buffer Protocol by handling negative dataLen (#1378)
- codec: Fix initial default values when using linear allocator (#1374)
- remote: Re-cap bytes in `defaultByteBuffer.Make` to correct capacity (#1361)
- grpc: Fix compressor `mcache.Free` panic when data is empty (#1364)
- grpc: Release connection in `DoFinish` for gRPC streaming to close short connections (#1328)

## Improvements

- loadbalance: Consistent hash no longer caches all keys; removes background expiry and singleflight; reduces memory (#1370)
- perf: Use `dirtmake.Bytes` in `defaultByteBuffer` to reduce memclr cost (#1314)
- codec/thrift: Fast codec uses batch allocation for struct fields (#1320)
