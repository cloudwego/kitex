## Breaking Changes

- ttstream: Move `pkg/remote/trans/ttstream/container` to `pkg/remote/trans/ttstream/internal/container` (#1952)
  Migration: external imports of the old path will fail to compile. The package was not intended as public API; copy the needed implementations into your own module if you depended on them.
- grpc: `grpc.NewClientTransport`'s `onClose`/`onGoAway` callbacks now fire after the transport transitions to closing/draining and after `http2Client.mu` is released; callers relying on the old ordering should migrate to `grpc.NewClientTransportWithConfig` (#1945)

## Deprecations

- client: Deprecate `client.WithMuxConnection(connNum int)`; netpollmux is no longer maintained (#1933)
- server: Deprecate `server.WithMuxTransport()`; netpollmux is no longer maintained (#1933)
- netpollmux: Mark the `pkg/remote/trans/netpollmux` package as deprecated (#1933)
- server: Deprecate `server.InvokeCaller`; use `server.LocalCaller` instead (#1930)
- grpc: Deprecate `grpc.NewClientTransport(ctx, conn, opts, remoteService, onGoAway, onClose)`; use `grpc.NewClientTransportWithConfig(ctx, conn, opts, grpc.ClientConfig)` instead. The deprecated entry point also has a callback-timing behavior change — see Breaking Changes (#1945)
- kerrors: Restore `kerrors.ErrRPCFinish` as deprecated; kept for backward compatibility with pre-v0.15 code (#1953)

## Features

- server: Add `server.LocalCaller` and `server.NewLocalCaller(caller string, svr Server) (LocalCaller, error)` for in-process unary calls that reuse the server middleware/tracing/recovery/RPCInfo; streaming methods are rejected (#1930)
- rpcinfo: Add `rpcinfo.NewRPCInfoWithInlineFields() RPCInfo` returning an RPCInfo with inlined sub-objects in a single pool entry, reducing per-request pool gets and allocations (#1935)
- generic: Add `callopt.WithBinaryGenericIDLService(svcName)` / `streamcall.WithBinaryGenericIDLService(svcName)` to set the target IDL Service Name per call; effective only for `BinaryThriftGenericV2` / `BinaryPbGeneric` (#1928)
- grpc: Add `grpc.ClientConfig` and `grpc.NewClientTransportWithConfig(ctx, conn, opts, ClientConfig)`; new `OnClose`/`OnGoAway` callbacks receive context and transport (#1945)
- grpc: Add `grpc.ReuseWriteBufferConfig.EnableReuseHTTP2FramerBuffer` and `grpcframe.Framer.SetWriteBufferPoolEnabled(enabled bool)` to pool the HTTP/2 framer write buffer via `mcache` (#1944)
- event: Add `event.GetDefaultEventNum()` / `event.SetDefaultEventNum(num int)` for tuning the default event queue capacity (default 200, affects new queues only) (#1939)

## Fixes

- ttstream: `Recv` timeout with `DisableCancelRemote=true` no longer recycles the long connection, so subsequent `Recv` calls can continue; stream timeout now produces a distinct `code=12015` exception (#1952)
- grpc: Fix connection pool leak when a connection is closed or receives GOAWAY with no subsequent calls; dead transports now self-remove from the pool slot via `onClose`/`onGoAway` callbacks (#1945)
- codec/thrift: `frugalAvailable` now returns true for void-func result structs that only contain `_unknownFields` (under `KeepUnknownFields`) so Frugal can encode/decode them (#1938)
- generic: Fix panic when writing a list/map with heterogeneous element types by resolving the writer per element instead of caching the first element's writer (#1926)
- streaming: Server-side old `Stream` wrapped in server middleware is no longer discarded (#1929)
- ttstream: Add server-side information to the `errBizHandlerReturnCancel` exception so cascaded callers can distinguish server- vs client-originated cancels (#1921)
- client: Close the ticker in the `rpctimeout` pool to prevent ticker leak (#1931)
- streaming: Remove unnecessary `rpcStats.Reset()` from `finishTracer` in nphttp2/ttstream server handlers (#1922)

## Improvements

- grpc: Reduce client-side object allocations on unified cancel scenarios by reusing pre-built `*status.Status` singletons for `context.Canceled`/`context.DeadlineExceeded` (#1950)
- discovery: Reduce object allocations in the discovery event queue (#1939)
- rpcinfo: Use inline RPCInfo fields on server hot paths to reduce per-request allocations (#1935, #1940)
- grpc: Pool the HTTP/2 framer write buffer via `mcache` (gated by `ReuseWriteBufferConfig.EnableReuseHTTP2FramerBuffer`) to reduce idle connection memory at the cost of per-write allocation/free overhead (#1944)
- loadbalance: Replace `github.com/bytedance/gopkg/util/xxhash3` with `hash/maphash` for consistent-hash key/node hashing in `consistBalancer`; the per-process random seed makes hash values differ across client replicas and restarts (#1924)
