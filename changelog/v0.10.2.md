## Breaking Changes

- remote: Change `NewMessageWithNewer` -- `svcSearchMap map[string]*serviceinfo.ServiceInfo` replaced by `svcSearcher ServiceSearcher` interface; `refuseTrafficWithoutServiceName` parameter removed (#1439)
  Migration: use new `ServiceSearcher` interface instead of map.
- remote: Replace `ServerOption.SvcSearchMap` with `ServerOption.SvcSearcher` of type `ServiceSearcher` (#1439)
- remote: Remove `ServerOption.RefuseTrafficWithoutServiceName` field (#1439)
  Migration: use `ServiceSearcher` strict mode instead.
- remote: Remove `BuildMultiServiceKey` function (#1439)
- reflection/thrift: Remove `pkg/reflection/thrift` package including `Files`, `GlobalFiles`, `NewFiles()`, `RegisterIDL()` (#1433)
  Migration: use alternative reflection mechanism if needed.

## Features

- remote: Add `ServiceSearcher` interface with `SearchService(svcName, methodName string, strict bool) *serviceinfo.ServiceInfo` (#1439)
- retry: Add `GenRetryKeyFunc` type and `WithCustomizeKeyFunc` option to customize retry policy lookup key (#1363)
- generic: Support returning `BizStatusError` for binary generic calls (#1404)
- tool: Implement kitex tool as SDK via `tool/cmd/kitex/sdk.RunKitexTool` and `sdk.GetKiteXSDKPlugin` (#1356)
- serviceinfo: Add `CombineService` and `CombineService_` constants (#1439)

## Fixes

- server: Fix "unknown service" error when using CombineService client by searching serviceInfo by method name (#1402)
- rpcinfo: Fix `SetTransportProtocol` for `PurePayload` -- now overrides all bits instead of OR-ing (#1436, #1438)
- generic/http: Fix payload length check for HTTP generic by gating dynamicgo path on non-zero `PayloadLen` (#1416)
- client: Fix business timeout detection in `makeTimeoutErr` to correctly identify biz timeout before checking deadline (#1392)

## Improvements

- loadbalance: Trigger rebalance when instance weights are updated, not just on add/delete events (#1397)
- discovery: `DefaultDiff` now detects weight changes as `Change.Updated` for rebalance (#1397)
