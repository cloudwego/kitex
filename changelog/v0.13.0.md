## Breaking Changes

- rpcinfo: Default transport protocol changed from `PurePayload` (buffered) to `transport.Framed` (#1723)
  - Before: `rpcConfig.transportProtocol = 0` (PurePayload/buffered)
  - After: `rpcConfig.transportProtocol = transport.Framed`
  - Migration: explicitly set `client.WithTransportProtocol(transport.PurePayload)` or `server.WithTransportProtocol(transport.PurePayload)` if buffered protocol is required.
- transport: Remove `STREAMING` constant; replace with `TTHeaderStreaming` and `GRPCStreaming` as separate flags; `Protocol.String()` returns pipe-separated composite (#1658)
  - Before: `transport.TTHeaderStreaming = TTHeader | STREAMING`
  - After: `transport.TTHeaderStreaming` is its own constant; new `transport.GRPCStreaming` added
  - Migration: replace `transport.STREAMING` with `transport.TTHeaderStreaming` or `transport.GRPCStreaming`.
- streaming: Deprecate `streaming.Stream` interface; replace with `streaming.ClientStream` and `streaming.ServerStream` (#1658)
  - Before: `Stream.SendMsg(m interface{}) error`
  - After: `ClientStream.SendMsg(ctx context.Context, m any) error`
  - Migration: switch to `streaming.ClientStream`/`streaming.ServerStream`.
- endpoint: Move `RecvEndpoint`, `RecvMiddleware`, `SendEndpoint`, `SendMiddleware` and related types to deprecated (#1658)
  Migration: use `cep.StreamRecvMiddleware` (client) or `sep.StreamRecvMiddleware` (server).
- pkg/streamx: Remove entire `pkg/streamx` package; streaming types consolidated into `pkg/streaming` (#1658, #1741)
  Migration: use `pkg/streaming` instead.
- client/streamxclient, server/streamxserver: Remove packages (#1658)
  Migration: use new streaming options on `client` and `server` packages.
- client: Move `client.Options.RetryMethodPolicies` to `client.Options.UnaryOptions.RetryMethodPolicies` (#1658)

## Deprecations

- generic: `generic.NewThriftContentProvider` deprecated in favor of `generic.NewThriftContentWithAbsIncludePathProvider` which supports both relative and absolute include path searches (#1740)
- generic: `generic.NewThriftContentProviderWithDynamicGo` deprecated in favor of `generic.NewThriftContentWithAbsIncludePathProviderWithDynamicGo` (#1740)
- server: `server.WithRecvMiddleware`, `server.WithRecvMiddlewareBuilder`, `server.WithSendMiddleware`, `server.WithSendMiddlewareBuilder` deprecated; use `server.WithStreamRecvMiddleware` / `server.WithStreamSendMiddleware` via `server.WithStreamOptions` instead (#1658)

## Features

- streaming: new `streaming.ClientStream` and `streaming.ServerStream` interfaces with context-aware `SendMsg`/`RecvMsg` and header/trailer support (#1658)
- streaming: `streaming.Args` and `streaming.Result` now carry `ServerStream` and `ClientStream` fields (#1658)
- streaming: `streaming.CloseCallbackRegister` interface for registering stream close callbacks (#1658)
- endpoint: new `pkg/endpoint/cep` (client endpoint) package with `cep.StreamEndpoint`, `cep.StreamMiddleware`, `cep.StreamRecvMiddleware`, `cep.StreamSendMiddleware` and builders (#1658)
- endpoint: new `pkg/endpoint/sep` (server endpoint) package with `sep.StreamEndpoint`, `sep.StreamMiddleware`, `sep.StreamRecvMiddleware`, `sep.StreamSendMiddleware` and builders (#1658)
- client: `client.WithStreamOptions(opts ...StreamOption)` for stream-only client options; includes `client.WithStreamRecvTimeout`, `client.WithStreamMiddleware`, `client.WithStreamRecvMiddleware`, `client.WithStreamSendMiddleware` and builder variants (#1658)
- client: `client.WithUnaryOptions(opts ...UnaryOption)` for unary-only client options; includes `client.WithUnaryRPCTimeout`, `client.WithUnaryMiddleware` (#1658)
- client: `client.WithTTHeaderStreamingOptions(opts ...TTHeaderStreamingOption)` for TTHeader streaming transport options (#1658)
- client: `client.TailOption(opt)` to defer option execution until after all other options are applied (#1724)
- client/callopt/streamcall: `streamcall.WithRecvTimeout(d)` call option for per-call recv timeout on TTHeader streaming (#1736)
- server: `server.WithStreamOptions(opts ...StreamOption)` for stream-only server options (#1658)
- server: `server.WithUnaryOptions(opts ...UnaryOption)` for unary-only server options (#1658)
- server: `server.WithTTHeaderStreamingOptions(opts ...TTHeaderStreamingOption)` for TTHeader streaming transport options (#1658)
- circuitbreak: `CBSuite.StreamingServiceCBMW()` returning `cep.StreamMiddleware` for streaming circuit breaking (#1658)
- circuitbreak: `circuitbreak.NewStreamCircuitBreakerMW(control, panel)` for creating streaming-specific circuit breaker middleware (#1658)
- grpc: `http2Client.Dump()` and `connPool.Dump()` for dumping gRPC client connection and stream status for debugging (#1702)
- codec/thrift: fallback to fastcodec/frugal even when `CodecType=Basic`, removing the requirement for explicit codec type configuration (#1723)
- tool: add `prutal` as a built-in protobuf code generator, replacing `protoc` dependency; `tool/internal_pkg/prutal.PrutalGen` and `tool/internal_pkg/tpl/pbtpl` package (#1719)
- tool: disable `Validate` and `DeepEqual` codegen by default to simplify generated code (#1711)
