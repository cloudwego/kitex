## Features

- grpc: Support reusing write buffer per connection via `grpc.ReuseWriteBufferConfig`; adds `reuseBufWriter` that allocates from `mcache` pool on-demand and frees after flush, trading CPU for reduced memory (~2*WriteBufferSize per idle connection) (#1918)
- client: Add `client.WithGRPCReuseWriteBuffer(grpc.ReuseWriteBufferConfig)` option (#1918)
- server: Add `server.WithGRPCReuseWriteBuffer(grpc.ReuseWriteBufferConfig)` option (#1918)

## Improvements

- ttstream: Remove dedicated write goroutine from both `clientTransport` and `serverTransport`; `WriteFrame` now encodes and flushes under a mutex instead of piping through a `container.Pipe[*Frame]`, eliminating the Sender OOM risk from unbounded frame buffering (#1917)
