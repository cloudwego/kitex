## Features

- generic: Add `WithDynamicGoOptions(opts *dthrift.Options)` ThriftIDLProviderOption to pass custom dynamicgo parsing options; add `DynamicGoOptions` field to `generic.ProviderOption` (#1878)
- tool: Execute custom templates per-service when `loop_service` is set to true, even without combine-service mode; populate `PackageInfo.Services` in both protoc and thriftgo plugins (#1893)

## Fixes

- retry: Fix `mixedRetryer.Do` waiting for backoff even when the first call succeeds; check `isRetryResult` before `ShouldRetry` to skip unnecessary backoff on success (#1879)
- ttstream: Send Header Frame in `serverStream.CloseSend` when handler exits without sending any message, preventing the client from hanging on `stream.Header()` (#1887)
- ttstream: Inject `K_METHOD` into server-side stream context so downstream RPC calls carry correct `FROM_METHOD` information (#1852)
- rpcinfo: Fix type conversion panic in `FreezeRPCInfo` when called twice by handling non-`*invocation` types via interface assertion in `copyInvocation` (#1888)
- tool: Move git-clone of remote IDL includes before prutal plugin invocation to ensure dependencies are available (#1900)

## Improvements

- grpc: Avoid grpc `cleanupStream` `onWrite` closure capturing `Stream` to reduce memory pressure; nil out `itemList.it` on dequeue in `controlbuf` (#1886)
- client: Add `to_service` info to connection failure auto-retry log for easier troubleshooting (#1880)
