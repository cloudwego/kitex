## Deprecations

- rpcinfo: `TraceController.GetStreamEventHandler()` is deprecated; use `TraceController.Handle*` methods instead (#1905)
- internal/stream: `StreamEventHandler` type is deprecated (exposed via `TraceController.GetStreamEventHandler()`); use `rpcinfo.ClientStreamEventHandler` or `rpcinfo.ServerStreamEventHandler` instead (#1905)
- client: `client.WithStreamRecvTimeout` is deprecated; use `client.WithStreamRecvTimeoutConfig` instead (#1911)
- client/callopt/streamcall: `streamcall.WithRecvTimeout` is deprecated; use `streamcall.WithRecvTimeoutConfig` instead (#1911)

## Features

- streaming/tracing: Add detailed streaming event tracing with new types `rpcinfo.ClientStreamEventHandler`, `rpcinfo.ServerStreamEventHandler`, `rpcinfo.StreamStartEvent`, `rpcinfo.StreamRecvEvent`, `rpcinfo.StreamSendEvent`, `rpcinfo.StreamRecvHeaderEvent`, `rpcinfo.StreamFinishEvent` and corresponding `TraceController.Handle*` methods (#1905)
- client: Add `client.WithStreamEventHandler(rpcinfo.ClientStreamEventHandler)` StreamOption for registering detailed streaming event handlers (#1905)
- server: Add `server.WithStreamEventHandler(rpcinfo.ServerStreamEventHandler)` StreamOption for registering detailed streaming event handlers (#1905)
- streaming: Add `streaming.TimeoutConfig` struct with `Timeout` and `DisableCancelRemote` fields for fine-grained Recv timeout control (#1911)
- client: Add `client.WithStreamRecvTimeoutConfig(streaming.TimeoutConfig)` StreamOption (#1911)
- client/callopt/streamcall: Add `streamcall.WithRecvTimeoutConfig(streaming.TimeoutConfig)` call option (#1911)
- rpcinfo: Add `StreamConfig.StreamRecvTimeoutConfig()` method and `MutableRPCConfig.SetStreamRecvTimeoutConfig(streaming.TimeoutConfig)` (#1911)
- stats: Add `stats.StreamStart`, `stats.StreamRecvHeader`, `stats.StreamFinish` events (#1905)
- codes: Add `codes.RecvDeadlineExceeded` (code 17) for stream Recv timeout errors (#1911)
- kerrors: Add `kerrors.ErrStreamingTimeout` error sentinel for streaming timeout errors (#1911)
- ttstream: Add `ttstream.WithClientTraceController(*rpcinfo.TraceController)` ClientHandlerOption for injecting trace controller (#1905)
- grpc: Add `TraceController` field to `grpc.ConnectOptions` for reporting detailed streaming events on client side (#1905)

## Fixes

- streaming: Disable rpcinfo reuse for streaming RPCs; users commonly launch goroutines that reference rpcinfo after handler returns, so usability takes precedence over pool performance (#1909)
- grpc/server: Change `http2Server.finishStream` cancel cause from nil to `errBizHandlerReturn` for better error diagnostics (#1911)
- grpc/server: Downgrade `loopyWriter.run` returning error log from Error to Info level since gRPC connection closures are expected with proxies/sidecars (#1911)
